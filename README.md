# FoodHub
An Exploratory Data Analysis (EDA) of New York City food aggregator data to optimize restaurant performance and delivery logistics using Python.

### 📌 Project Overview

This project performs an Exploratory Data Analysis (EDA) for **FoodHub**, a food aggregator company that connects customers with multiple restaurants in New York City via a single app. The goal is to analyze customer demand, restaurant performance, and delivery logistics to help improve the overall business strategy and customer experience.

### 📊 Dataset Description

The dataset contains **1,898 orders** and **9 features**, including:

- **Order Details:** Unique Order ID and Customer ID.
- **Restaurant Info:** Restaurant name and cuisine type.
- **Financials:** Cost of the order.
- **Logistics:** Food preparation time and delivery time (in minutes).
- **Ratings:** Ratings provided by customers (on a scale of 1 to 5).

### 🛠️ Tech Stack & Libraries

The following Python libraries were used for data cleaning, analysis, and visualization:

- **Data Manipulation:** `Pandas`, `NumPy`.
- **Data Visualization:** `Matplotlib`, `Seaborn`.

### 🔍 Key Insights from EDA

- **Preparation Time:** On average, it takes approximately **27.37 minutes** for food to be prepared, with a minimum of 20 minutes and a maximum of 35 minutes.
- **Delivery Performance:** Delivery times range from 15 to 33 minutes.
- **Customer Feedback:** About **38.78%** of orders did not receive a rating, representing a significant area for potential feedback improvement.
